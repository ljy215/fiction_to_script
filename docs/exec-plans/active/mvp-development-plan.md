# MVP 开发计划

版本：v1.0  
日期：2026-06-05  
状态：Active

## 1. 目标

开发一个 AI 小说转剧本 Web 应用 MVP。用户登录后可以导入至少 3 个章节的多语言小说，选择剧本类型，通过 LangGraph 编排的智能体流程调用阿里百炼 API，生成一个完整中文 YAML 剧本，在剧本视图编辑器中继续修改，保存历史版本，局部重生成，并导出 YAML。

## 2. 已确认技术决策

- 前端：Vite + React + JavaScript。
- 前端包管理：npm。
- 后端：Python + FastAPI。
- 后端依赖管理：uv 为主，同时维护 `requirements.txt`，方便 pip 方式复现。
- 数据库：SQLite。
- ORM：SQLAlchemy。
- 登录鉴权：JWT。
- 智能体编排：LangGraph。
- 首期模型供应商：阿里百炼 API。
- 文件存储：本地磁盘。
- 长任务：FastAPI BackgroundTasks + 数据库任务状态轮询。
- 启动方式：需要 Docker 一键启动前端、后端和 SQLite 数据目录挂载。
- 导出格式：只导出 YAML。

## 3. 开发原则

- 每个 PR 只做一件事。
- 每个 PR 合并后主分支必须可运行。
- 大功能拆成小 PR，避免一个 PR 混合前端、后端、AI、编辑器和导出等多个主题。
- 行为、数据结构、目录结构、启动方式发生变化时，必须同步更新文档。
- 涉及阿里百炼 API 的 PR 必须避免泄露 API Key。
- 涉及 YAML 的 PR 必须保持与 `docs/script-yaml-schema.md` 一致。

## 8. 完成定义

每个 PR 完成时必须满足：

- 只实现一个清晰范围。
- 主分支仍可运行。
- 必要文档已更新。
- 有明确测试方式。
- 不泄露 API Key、JWT 密钥、用户密码。
- 不引入无关重构。

MVP 完成时必须满足：

- 用户能登录。
- 用户能创建项目。
- 用户能导入至少 3 章小说。
- 用户能选择剧本类型。
- 系统能生成完整中文 YAML 剧本。
- 用户能在剧本视图编辑器中修改。
- 用户能保存历史版本。
- 用户能局部重生成。
- 用户能导出 YAML。
- Docker 能启动可演示环境。
